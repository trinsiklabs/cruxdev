# User Journey Maps

> Visitor paths through the website, mapped per audience segment.

## 1. Journey Map: [Primary Audience Name]

**Persona:** [e.g., "Technical Founder — Sarah, 32, CTO of a seed-stage startup"]
**Goal:** [e.g., "Evaluate whether this product solves her infrastructure problem"]
**Entry point:** [e.g., "Google search for 'automated deployment tool for startups'"]

### Journey Stages

| Stage | Page(s) | What They Think | What They Do | What They Feel | Success Metric |
|---|---|---|---|---|---|
| **Awareness** | [Homepage or blog post via search] | "Does this solve my problem?" | Scans headline, reads value prop | Curious but skeptical | [Time on page > 30s] |
| **Interest** | [Features page] | "How does this actually work?" | Reads feature details, watches demo | Interested, comparing mentally | [Scroll depth > 75%] |
| **Evaluation** | [Pricing page] | "Can I afford this? Is it worth it?" | Compares plans, checks competitors | Calculating, looking for deal-breakers | [Pricing page visit] |
| **Decision** | [Case study or testimonial] | "Has this worked for someone like me?" | Reads case study, checks credibility | Building confidence | [Case study click] |
| **Conversion** | [Signup / demo page] | "Let me try this" | Clicks CTA, fills out form | Committed, wants quick start | [Form completion] |

### Key Friction Points

| Stage | Friction | Mitigation |
|---|---|---|
| [e.g., Awareness] | [e.g., "Value prop too vague — doesn't say what the product IS"] | [e.g., "Lead with concrete benefit, not abstract language"] |
| [e.g., Evaluation] | [e.g., "No pricing visible — user bounces to competitor"] | [e.g., "Transparent pricing on dedicated page"] |
| [e.g., Conversion] | [e.g., "Signup form asks for too much info"] | [e.g., "Reduce to email + password only"] |

### Ideal Path (Happy Path)

```
Google Search → Blog Post → Features Page → Pricing → Demo Signup
                    ↓
              Homepage (direct) → Features → Case Study → Demo Signup
```
