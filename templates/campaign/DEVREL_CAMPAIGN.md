### Content Rules for Developer Audience

| Do | Don't |
|---|---|
| Show real code, real problems, real solutions | Use marketing language in technical content |
| Be honest about limitations and trade-offs | Claim to be "the best" without evidence |
| Provide working code examples they can run | Show fake/simplified code that doesn't work |
| Link to documentation, not marketing pages | Gate technical content behind email forms |
| Use their terminology, not our marketing terms | Oversimplify for the sake of conversion |

| [Twitter/X community] | Twitter | [X followers] | [Active / New] | [Threads, conversations, help] |

### Engagement Rules

1. **Give value first** — answer questions, help people, share knowledge
2. **Be transparent** — always disclose affiliation with our product
3. **Don't spam** — one mention per thread max, only when genuinely relevant
4. **Be patient** — community trust takes months to build
5. **Accept criticism** — developers respect honesty, not defensiveness
