# CruxDev Vertical Expansion Research

**Date:** 2026-03-28
**Status:** Research complete — product strategy recommendation
**Method:** Web research across 20+ search queries, scored against CruxDev convergence model fit

---

## Executive Summary

CruxDev's autonomous convergence model — iterative audit-fix-re-audit loops with mathematically defined "done" — is a strong fit for verticals where:
1. Quality has measurable dimensions (not purely subjective)
2. Work involves multiple interdependent artifacts (plans, content, assets, configs)
3. Iteration demonstrably improves output (convergence is valuable, not just completion)
4. The work is currently done ad-hoc with AI tools but lacks orchestration

The biggest opportunity is in verticals where people are already using AI heavily but have NO convergence framework — they run one-shot prompts and accept whatever comes out. CruxDev's value proposition is turning that into systematic, audited, converged output.

## Gaps CruxDev Must Close

1. **Domain-specific audit dimensions:** The current audit system is code-focused (8 code + 5 doc dimensions). Non-software verticals need custom audit dimension registries.

2. **Non-code artifact auditing:** Most new verticals produce documents, spreadsheets, images, audio, and video — not source code. The convergence engine needs to audit these artifact types.

3. **Quantitative metric integration:** E-commerce, marketing, fitness, and supply chain verticals require pulling live data (analytics, performance metrics) into the audit loop.

4. **Template marketplace:** With 20+ verticals, templates become a key differentiator. Need a template registry that's easy to browse and customize.

5. **Industry-specific pattern docs:** Each vertical needs its own methodology doc (like DEVELOPMENT_PATTERNS_CRUXDEV.md but for that vertical).

6. **Compliance/regulatory framework:** Insurance, recruiting, and real estate verticals have legal requirements that must be tracked as first-class audit dimensions.
